# Fix: Авто-экспонаты (20+) — замена пробы файлов на манифест

## Проблема
`scanAdditionalExhibits` пробует файлы `new Image()` — ненадёжно на Vercel CDN. Замена: манифестный JSON + `fetch()`.

## План

### 1. Создать `assets/exhibits/manifest.json`
Содержит массив объектов для экспонатов, которых нет в inline JSON (20-71):
```json
[
  { "file": "exhibit-20.webp" },
  { "file": "exhibit-21.webp" },
  ...
  { "file": "exhibit-71.webp" }
]
```

### 2. Переписать `script.js`
- **Удалить** `scanAdditionalExhibits` (строки 257-351)
- **Добавить** `loadExhibitManifest()`:
  - `fetch("assets/exhibits/manifest.json")`
  - Собрать `describedSrcs` из inline JSON (`#gallery-data`)
  - Пропустить файлы, уже описанные в JSON
  - Для каждого нового файла — создать `gallery-item.no-info.has-lightbox.animate-up` с `img` + lightbox-обработчиками
  - По завершении — `initAnimateObserver(grid)`
- **Заменить** вызов `scanAdditionalExhibits(...)` на `loadExhibitManifest()` (строка 371)

### 3. Убрать scan-indicator из `gallery.html`
Удалить строки 215-220 (`<div id="scan-indicator">...</div>`)

## Файлы
| Файл | Действие |
|------|----------|
| `assets/exhibits/manifest.json` | Создать |
| `script.js` | Заменить `scanAdditionalExhibits` на `loadExhibitManifest` |
| `gallery.html` | Удалить `scan-indicator` div |

## Верификация
1. `gallery.html` локально — все 71 экспонат отображаются
2. `index.html` — стили подключены
3. Деплой на Vercel — `fetch` манифеста работает, экспонаты грузятся

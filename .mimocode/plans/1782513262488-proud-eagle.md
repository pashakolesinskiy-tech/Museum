# Fix: Авто-экспонаты (20+) не загружаются на Vercel

## Проблема
Функция `scanAdditionalExhibits` в `script.js:257-351` автоматически обнаруживает дополнительные экспонаты (exhibit-20..71) путём последовательного пробы файлов через `new Image()`. На Vercel CDN задержка ответа на первый запрос к edge может превышать 3-секундный таймаут (`script.js:293`), после чего 2 подряд пропуска — и сканирование прекращается (`script.js:281`).

## Дополнительный баг
В `index.html:62` сломанный HTML-комментарий поглощает подключение `style.css`:
```html
<!--link rel="stylesheet" href="style.css" />
```

## План исправлений

### 1. Увеличить таймаут и добавить повтор (`script.js`)
- **Файл**: `script.js`, строки 278-299
- Увеличить таймаут с 3000мс до 8000мс
- Увеличить порог остановки с 2 до 4 пропусков (`script.js:281`)

### 2. Исправить HTML-комментарий (`index.html`)
- **Файл**: `index.html`, строка 62
- Заменить `<!--link rel="stylesheet" href="style.css" />` на `<link rel="stylesheet" href="style.css" />`

## Верификация
1. Открыть `gallery.html` локально → экспонаты 20-71 должны отображаться
2. Проверить `index.html` → стили должны применяться
3. После деплоя на Vercel → дополнительные экспонаты должны подгружаться

# Подробный технический отчёт о веб-сайте музея

## 1. Общее описание проекта

Виртуальный музей Государственного учреждения образования «Детский сад №24 имени П.С. Портновой г. Полоцка» представляет собой интерактивную веб-экспозицию, посвящённую истории дошкольного учреждения и его основательнице — Полине Сергеевне Портновой. Сайт создан с целью сохранения памяти о героическом прошлом, демонстрации архивных материалов, фотографий и видеозаписей, а также предоставления возможности виртуального посещения музея.

Проект выполнен в виде многостраничного веб-сайта с адаптивным дизайном, поддержкой анимаций и интерактивных элементов. Сайт использует современные веб-технологии без использования фреймворков, что обеспечивает высокую производительность и простоту развёртывания.

## 2. Структура файлов проекта

### 2.1 Основные файлы

Корневая директория проекта содержит следующие основные файлы:

- `index.html` — главная страница сайта
- `gallery.html` — страница галереи экспонатов
- `portnova.html` — страница биографии П.С. Портновой
- `video-gallery.html` — страница видео-галереи
- `style.css` — главный файл стилей
- `script.js` — главный файл JavaScript
- `README.md` — документация проекта

### 2.2 Директория assets

Все медиа-ресурсы расположены в директории `assets/`:

| Директория | Содержимое | Количество |
|------------|------------|------------|
| `carusel/` | Изображения для 3D-карусели на главной странице | 3 файла |
| `exhibits/` | Фотографии экспонатов галереи | 19 файлов |
| `portnova/` | Архивные фотографии П.С. Портновой | 15 файлов |
| `bw/img/` | Чёрно-белые изображения для видео-галереи | 7 файлов |
| `bw/video/` | Видеофайлы для видео-галереи | 5 файлов |
| Корневая директория | Основные изображения (фон, портреты) | 9 файлов |

### 2.3 Общий объём медиа-ресурсов

Суммарный объём всех медиа-ресурсов составляет приблизительно 50-60 МБ, что является значительным показателем для статического сайта и требует оптимизации загрузки.

## 3. Анализ HTML-структуры

### 3.1 Главная страница (index.html)

Главная страница содержит следующие основные секции:

**Хедер (header)** — фиксированная навигационная панель с логотипом и меню. Навигация включает ссылки на разделы: «Главная», «Экспозиции», «Связь времён», «Играй и познавай», «Контакты». Реализована поддержка мобильного меню через бургер-иконку.

**Секция hero** — главный баннер страницы с заголовком-цитатой «Эти стены помнят не только детский смех, на них печать нашего героического прошлого» и подзаголовком с указанием названия учреждения. Присутствует кнопка «Об экспозиции», открывающая модальное окно.

**Модальное окно «О музее»** — содержит развёрнутую информацию об истории детского сада, деятельности П.С. Портновой в годы Великой Отечественной войны, эвакуации 104 детей в город Шумерля, а также информацию о переданных родственниками музейных экспонатах.

**Секция карусели (carousel-section)** — трёхмерная карусель с тремя карточками-ссылками на основные разделы: «Всё ради жизни на земле» (portnova.html), «Детство в деталях» (gallery.html), «Через огненные вёрсты войны».

**Секция «Связь времён»** — галерея-слайдер с тремя историческими фотографиями, включая портрет П.С. Портновой. Первое изображение имеет атрибут `data-video-src` для воспроизведения видео в модальном окне.

**Секция «Играй и познавай»** — сетка из трёх карточек с изображениями в стиле оконных рам, представляющих интерактивный раздел для посетителей.

**Футер** — контактная информация: название организации, дата создания (2026), email, телефон, ссылки на социальные сети (VK, Telegram, YouTube).

**Модальное окно для видео** — скрытый элемент, активируемый при клике на изображения с видео-контентом.

### 3.2 Страница галереи (gallery.html)

Страница галереи экспонатов содержит:

- Упрощённый хедер с навигацией (Главная, Видео-галерея)
- Заголовок с описательной подписью о советских игрушках как «особом языке искренности и простоты»
- Сетку из 19 карточек экспонатов с фотографиями и информацией

**Коллекция экспонатов** включает предметы разных периодов:

- Резиновые игрушки («Карлсон», «Птичка»)
- Заводные механические игрушки («Петушок», «Курочка»)
- Куклы различных модификаций («Пупсик», «Школьница», винтажная кукла)
- Детский утюг
- Каталки («Петушок», «Лисичка»)
- Целлулоидная «Неваляшка»
- Детская швейная машинка (производство города Орша)

Каждый экспонат сопровождается описанием и датой выпуска (преимущественно 1960-1980-е годы).

**Аудиоплеер** — на странице присутствует скрытый аудиоэлемент с фоновой музыкой (game.mp3), которая запускается после первого взаимодействия пользователя со страницей.

### 3.3 Страница биографии (portnova.html)

Персональная страница, посвящённая Полине Сергеевне Портновой:

- Портретное изображение в декоративной рамке
- Имя и даты жизни (02.05.1903 — 20.11.1989)
- Поэтическое посвящение из четырёх четверостиший
- Фотогалерея из 15 архивных изображений

Страница содержит собственный аудиофайл (back2.mp3) для фонового сопровождения.

### 3.4 Страница видео-галереи (video-gallery.html)

Специализированная страница с интерактивными видеоматериалами:

- Описание концепции: «каждое видео начинает свой рассказ с чёрно-белого кадра, а затем раскрывается в цвете»
- Вертикальный список из 7 элементов
- 4 видеофрагмента с автовоспроизведением по клику
- 2 статических изображения с эффектом раскрытия цвета (color-reveal)
- 1 видео без цветового перехода

Видеоматериалы охватывают период с 1943 по 1981 год, включая кадры из эвакуации в городе Шумерля.

## 4. Анализ CSS-стилей

### 4.1 Дизайн-система

Проект использует CSS-переменные для обеспечения консистентности стилей:

```css
:root {
  --bg: #060709;                    /* Тёмный фон */
  --surface-light: rgba(255, 255, 255, 0.03);
  --light: #f8f5f2;
  --text: #f5f7fa;
  --text-muted: rgba(245, 247, 250, 0.75);
  --accent: #d4af37;                 /* Золотой акцент */
  --accent-soft: rgba(212, 175, 55, 0.15);
  --accent-glow: rgba(212, 175, 55, 0.3);
  
  --font-serif: "Cormorant Garamond", serif;
  --font-sans: "Outfit", sans-serif;
  
  --radius: 24px;
  --radius-sm: 18px;
  --transition: 0.4s cubic-bezier(0.2, 0.9, 0.3, 1);
  --shadow: 0 25px 50px -12px rgba(0, 0, 0, 0.6);
  
  --header-height: 80px;
}
```

### 4.2 Типографика

Используются два шрифта из Google Fonts:

- **Cormorant Garamond** — элегантный антиквенный шрифт для заголовков, создающий атмосферу музейной торжественности
- **Outfit** — современный гротеск для основного текста, обеспечивающий хорошую читаемость

### 4.3 Цветовая схема

Проект выполнен в «дорогом» тёмном режиме с золотыми акцентами:

- Основной фон: тёмно-серый (#060709)
- Текст: светло-серый (#f5f7fa)
- Акцентный цвет: золотой (#d4af37)
- Фоновое изображение: Background.webp с градиентным оверлеем

### 4.4 Адаптивность

Реализована поддержка различных устройств:

- Мобильные устройства (до 480px)
- Планшеты (до 768px)
- Десктопы (более 768px)

Используются относительные единицы измерения (clamp, vw, vh) и медиа-запросы для гибкой верстки.

### 4.5 Эффекты и анимации

**Glassmorphism** — эффект матового стекла в навигационном меню:

```css
.header {
  background: rgba(10, 15, 24, 0.3);
  backdrop-filter: blur(16px);
  -webkit-backdrop-filter: blur(16px);
}
```

**Анимация появления (animate-up)** — плавное всплывание элементов снизу вверх при прокрутке страницы:

```css
.animate-up {
  opacity: 0;
  transform: translateY(30px);
  transition: opacity 0.6s ease, transform 0.6s ease;
}

.animate-up.visible {
  opacity: 1;
  transform: translateY(0);
}
```

**Интерактивные эффекты карточек:**

- Градиентное затемнение
- Плавное увеличение (scale) при наведении
- Подсветка бортов золотым цветом

**Адаптация под iOS Safari** — специальныйфикс для проблем с background-attachment: fixed:

```css
@supports (-webkit-touch-callout: none) {
  body {
    background-attachment: scroll;
  }
}
```

## 5. Анализ JavaScript-функциональности

### 5.1 Бургер-меню

Реализация мобильной навигации с анимацией превращения в крестик:

- Переключение класса .active при клике
- Блокировка/разблокировка скролла страницы при открытом меню
- Автоматическое закрытие меню при клике на ссылку
- Закрытие меню при клике вне его области

### 5.2 Слайдер «Связь времён»

Простой горизонтальный слайдер с:

- Динамической генерацией точек навигации (dots)
- Кнопками «Предыдущее» и «Следующее»
- Бесконечной прокруткой (зацикливание)
- Анимацией перехода transform: translateX

### 5.3 Трёхмерная карусель

Сложный компонент с использованием CSS 3D-transforms:

- Предварительный расчёт позиций для оптимизации производительности
- Поддержка различных радиусов в зависимости от ширины экрана
- Адаптивные параметры: 200px (мобильные), 280px (планшеты), 350px (десктопы)
- Поддержка свайпов и drag-событий для мобильных устройств
- Различение кликов и перетаскиваний (didDrag flag)
- Навигация по точкам

### 5.4 Модальные окна

**Модальное окно «О музее»:**

- Открытие/закрытие по клику на кнопку
- Закрытие по клику на фон или клавише Escape
- Блокировка скролла страницы при открытии

**Модальное окно для видео:**

- Динамическая загрузка источника видео
- Автоматическое воспроизведение
- Остановка и сброс при закрытии

### 5.5 Интерактивные видео в video-gallery.html

Два режима работы:

**Режим «цветная картинка»** — при клике на контейнер переключается класс .playing, отображающий цветную версию изображения вместо чёрно-белой.

**Режим «видео»** — при клике начинается воспроизведение видео, при окончании видео контейнер возвращается в исходное состояние.

### 5.6 Анимация при скролле

Использование Intersection Observer API для производительной анимации:

```javascript
const observer = new IntersectionObserver(
  (entries) => {
    entries.forEach((entry) => {
      if (entry.isIntersecting) {
        entry.target.classList.add("visible");
        observer.unobserve(entry.target); // Анимация запускается один раз
      }
    });
  },
  { threshold: 0.1, rootMargin: "0px 0px -50px 0px" }
);
```

### 5.7 Фоновая музыка

Автоматическое воспроизведение аудио после первого взаимодействия пользователя:

- Установка громкости 30% для комфортного прослушивания
- Обработка ошибок автовоспроизведения (политика браузеров)
- Удаление обработчиков после успешного запуска

### 11.1 Производительность

Оптимизация производительности является критически важным аспектом для веб-сайта с большим объёмом медиаресурсов. Суммарный размер медиафайлов проекта составляет приблизительно 50-60 МБ, что создаёт значительную нагрузку на пропускную способность и время загрузки страниц.

#### Сжатие изображений в формате WebP

Рекомендуется конвертировать все существующие изображения в формат WebP, который обеспечивает значительно меньший размер файлов при сопоставимом качестве визуального отображения. Среднее сжатие при переводе JPEG в WebP составляет 25-35% без видимых потерь качества. Для изображений экспонатов в галерее (assets/exhibits/) и архивных фотографий П.С. Портновой (assets/portnova/) преобразование в WebP позволит существенно ускорить загрузку страниц, особенно на мобильных устройствах с ограниченной скоростью соединения. При конвертации следует использовать параметр качества 80-85% для фотографий и 90% для изображений с текстом или мелкими деталями. Необходимо сохранить оригинальные файлы JPEG в качестве резервной копии на случай, если某些 браузеры не поддерживают WebP.

#### Оптимизация видеоконтента

Видеофайлы занимают значительную часть общего объёма медиаресурсов проекта. Файл assets/video.mp4 имеет размер более 23 МБ, что существенно замедляет загрузку страницы. Рекомендуется использовать формат H.264 с адаптивным битрейтом: для мобильных устройств — 720p с битрейтом 1-2 Мбит/с, для десктопов — 1080p с битрейтом 3-5 Мбит/с. Для видеогалереи (assets/bw/video/) следует рассмотреть возможность использования более современных кодеков, таких как AV1, который обеспечивает лучшее сжатие по сравнению с H.264. Также рекомендуется добавить атрибут `preload="metadata"` для тегов video, чтобы браузер загружал только метаданные видео, а не весь файл целиком.

#### Ленивая загрузка (Lazy Loading)

В настоящее время на страницах сайта используются отдельные случаи атрибута loading="lazy", однако рекомендуется расширить его применение на все изображения, не находящиеся в области первоначального просмотра (above the fold). Для галереи экспонатов (gallery.html) с 19 изображениями ленивая загрузка позволит загружать изображения по мере прокрутки страницы пользователем, что сократит начальное время загрузки. Следует использовать нативный атрибут loading="lazy" для img-элементов, а для фоновых изображений — API Intersection Observer. При реализации ленивой загрузки необходимо предусмотреть placeholder-изображения или скелетонные загрузчики для визуального комфорта пользователей.

#### Оптимизация кодов состояния HTTP

Для повышения производительности рекомендуется настроить корректные коды состояния HTTP и заголовки кэширования. Статические ресурсы (изображения, CSS, JavaScript) должны иметь заголовок Cache-Control с длительным сроком кэширования (например, Cache-Control: public, max-age=31536000). Для HTML-страниц следует использовать заголовки ETag или Last-Modified для эффективной проверки изменений. Рекомендуется включить сжатие на стороне сервера (gzip или Brotli) для текстовых ресурсов, что может сократить размер CSS-файла (~40 КБ) и JavaScript-файла (~12 КБ) на 60-80%.

### 11.3 UX/UI-улучшения

Улучшение пользовательского опыта и интерфейса позволит повысить вовлечённость посетителей и сделать взаимодействие с сайтом более комфортным.

#### Форма обратной связи

На сайте отсутствует форма обратной связи, что ограничивает возможность коммуникации с посетителями. Рекомендуется добавить форму на главную страницу или создать отдельную страницу контактов. Форма должна включать следующие поля: имя пользователя (обязательное), email (обязательное, с валидацией формата), тема сообщения (выпадающий список или текстовое поле), текст сообщения (многострочное текстовое поле), опционально — чекбокс для согласия на обработку персональных данных в соответствии с GDPR и белорусским законодательством о защите персональных данных.

Реализацию формы рекомендуется выполнить с использованием JavaScript для клиентской валидации и отправки данных на сервер через AJAX-запрос. В качестве резервного варианта можно использовать сервисы форм, такие как Formspree или Netlify Forms, которые не требуют серверной части. При отправке формы необходимо обеспечить визуальную обратную связь: индикатор загрузки, сообщение об успешной отправке или об ошибке.

### 11.4 Accessibility (Доступность)

Обеспечение доступности веб-контента для пользователей с ограниченными возможностями является не только этическим требованием, но и технической необходимостью для расширения аудитории сайта.

#### ARIA-атрибуты

На сайте уже используются базовые ARIA-атрибуты для модальных окон, однако их применение требует расширения. Рекомендуется добавить следующие атрибуты: aria-label для кнопок без текстового содержимого (бургер-меню, кнопки закрытия), aria-expanded для интерактивных элементов, раскрывающих контент, aria-hidden для декоративных элементов, не несущих смысловой нагрузки, aria-current="page" для активного пункта навигации, role="navigation", role="main", role="footer" для семантических секций страницы.

Для модальных окон необходимо реализовать управление фокусом: при открытии модального окна фокус должен перемещаться на первый интерактивный элемент, при закрытии — возвращаться на элемент, вызвавший открытие. Следует также добавить обработку клавиши Escape для закрытия модальных окон.

#### Контрастность

Текущая цветовая схема сайта имеет достаточный уровень контрастности для основного текста (светло-серый текст на тёмном фоне), однако рекомендуется проверить контрастность всех элементов с использованием инструментов типа WCAG Contrast Checker. Для текста размером менее 18px (14px для жирного текста) минимальное соотношение контрастности должно составлять 4.5:1. Для большего текста и графических элементов интерфейса допускается соотношение 3:1. Следует также убедиться, что цветовая схема не является единственным способом передачи информации — все состояния (активный, hover, фокус) должны иметь дополнительные визуальные индикаторы.

#### Клавиатурная навигация

Все интерактивные элементы сайта должны быть доступны и управляемы с клавиатуры. Текущая реализация включает базовую поддержку, но рекомендуется проверить и улучшить следующие аспекты: порядок фокуса (Tab order) должен соответствовать визуальному порядку элементов на странице, все кнопки и ссылки должны иметь видимый индикатор фокуса (outline), трёхмерная карусель должна поддерживать управление с клавиатуры (стрелки влево/вправо), слайдер «Связь времён» должен иметь клавиатурное управление. Для улучшения клавиатурной навигации рекомендуется использовать CSS-свойство :focus-visible для отображения фокуса только при навигации с клавиатуры, а не при клике мышью.

### 11.5 PWA-функциональность

Преобразование сайта в Progressive Web Application (PWA) позволит пользователям устанавливать его на устройства и использовать в офлайн-режиме, что особенно актуально для музея, где посетители могут находиться в зонах с нестабильным интернет-соединением.

#### Манифест приложения

Необходимо создать файл manifest.json, содержащий метаданные приложения:

```json
{
  "name": "Виртуальный музей Детский сад №24",
  "short_name": "Музей Портновой",
  "description": "Виртуальный музей, посвящённый истории дошкольного учреждения",
  "start_url": "/",
  "display": "standalone",
  "background_color": "#060709",
  "theme_color": "#d4af37",
  "icons": [
    {
      "src": "/assets/icons/icon-192.webp",
      "sizes": "192x192",
      "type": "image/png"
    },
    {
      "src": "/assets/icons/icon-512.webp",
      "sizes": "512x512",
      "type": "image/png"
    }
  ]
}
```

Манифест необходимо подключить в HTML-файлах с помощью тега `<link rel="manifest" href="/manifest.json">`. Также следует подготовить иконки различных размеров (72, 96, 128, 144, 152, 192, 384, 512 пикселей) для различных устройств.

#### Service Worker

Service Worker обеспечивает кэширование ресурсов для офлайн-доступа. Рекомендуется реализовать стратегию кэширования с приоритетом использования кэша (Stale-While-Revalidate) для статических ресурсов:

```javascript
// sw.js
const CACHE_NAME = 'museum-v1';
const urlsToCache = [
  '/',
  '/index.html',
  '/gallery.html',
  '/portnova.html',
  '/video-gallery.html',
  '/style.css',
  '/script.js'
];

self.addEventListener('install', (event) => {
  event.waitUntil(
    caches.open(CACHE_NAME)
      .then((cache) => cache.addAll(urlsToCache))
  );
});

self.addEventListener('fetch', (event) => {
  event.respondWith(
    caches.match(event.request)
      .then((response) => {
        if (response) {
          return response;
        }
        return fetch(event.request).then((response) => {
          if (!response || response.status !== 200 || response.type !== 'basic') {
            return response;
          }
          const responseToCache = response.clone();
          caches.open(CACHE_NAME)
            .then((cache) => {
              cache.put(event.request, responseToCache);
            });
          return response;
        });
      })
  );
});
```

#### Офлайн-режим

При отсутствии интернет-соединения пользователи должны видеть информативную страницу офлайн-режима. Для этого можно создать специальную страницу, которая будет отображаться при ошибках сети. Также рекомендуется добавить визуальный индикатор статуса соединения (онлайн/офлайн) в интерфейс сайта.

### 11.6 Безопасность

Обеспечение безопасности веб-сайта критически важно для защиты данных пользователей и поддержания доверия к ресурсу.

#### HTTPS

Все современные веб-сайты должны использовать HTTPS-протокол. Для получения бесплатного SSL/TLS-сертификата рекомендуется использовать Let's Encrypt или аналогичные сервисы. При настройке HTTPS следует включить HTTP Strict Transport Security (HSTS), который предотвращает понижение уровня защиты до HTTP. Конфигурация должна включать: автоматическое перенаправление с HTTP на HTTPS, использование TLS версии 1.2 или выше, отключение устаревших протоколов (SSLv3, TLS 1.0, TLS 1.1).

#### Заголовки безопасности

Рекомендуется настроить следующие HTTP-заголовки безопасности на стороне сервера: Content-Security-Policy (CSP) для ограничения источников загружаемого контента и предотвращения XSS-атак, X-Content-Type-Options со значением "nosniff" для предотвращения MIME-type sniffing, X-Frame-Options со значением "DENY" или "SAMEORIGIN" для защиты от clickjacking-атак, Referrer-Policy для контроля передачи информации о referrer, Permissions-Policy для ограничения функциональности браузера.

Пример настройки CSP для сайта:

```
Content-Security-Policy: 
  default-src 'self'; 
  script-src 'self' 'unsafe-inline' https://fonts.googleapis.com; 
  style-src 'self' 'unsafe-inline' https://fonts.googleapis.com; 
  font-src 'self' https://fonts.gstatic.com; 
  img-src 'self' data: https:; 
  media-src 'self' https:; 
  connect-src 'self';
```

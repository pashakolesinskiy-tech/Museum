# Отчёт оптимизации кода — Виртуальный музей детского сада №24

> Дата: 04.04.2026

---

## Обзор

Проведён комплексный аудит и оптимизация кода статического сайта виртуального музея. Проект состоит из 8 HTML-страниц, 1 CSS-файла (3102 строки), 1 JS-файла (991 строка) и ~100 медиафайлов.

---

## Выполненные изменения

### 1. Удаление дублирующихся CSS-правил

**Файл:** `style.css`

В блоке `@media (max-width: 480px)` обнаружены полностью дублирующиеся правила для классов `.bio-hero`, `.bio-photo-grid`, `.bio-poem`, `.bio-poem-stanza p`, `.bio-dedication`, `.bio-dedication-line` (строки 1949–1975). Все эти правила уже были определены выше (строки 1899–1928).

**Результат:** Удалено **27 строк** дублирующегося CSS.

### 2. Удаление неиспользуемых CSS-классов

**Файл:** `style.css`

Классы `.window-arch` и `.window-shine` определены с `display: none` и не несут функциональной нагрузки — они присутствуют в HTML как `<div aria-hidden="true">`, но не отображаются и не переключаются через JS.

**Результат:** Удалено **8 строк** мёртвого CSS.

### 3. Удаление пустых записей экспонатов

**Файл:** `gallery.html`

Из 58 записей JSON-массива только 19 содержат осмысленные данные (name, alt, date). Остальные 39 записей (exhibit-20, exhibit-22–34, exhibit-36–58) имели пустые поля, что приводило к рендеру пустых карточек без подписей и без accessibility-меток.

**Результат:** Удалено **39 пустых записей** (~195 строк JSON). Галерея теперь содержит только 19 реальных экспонатов.

### 4. Удаление мёртвой регистрации Service Worker

**Файл:** `script.js`

Код регистрации Service Worker (`/sw.js`) присутствовал, но сам файл `sw.js` не существует в проекте. Это вызывало ошибку 404 при каждой загрузке страницы.

**Результат:** Удалено **8 строк** мёртвого JS-кода.

### 5. Удаление закомментированного HTML

**Файл:** `war-path.html`

В файле присутствовали крупные закомментированные блоки (~150 строк):
- Цитата Портновой П.С.
- Блок «Ключевые факты» с цифрами
- Полная секция «Хроника событий» (timeline)
- Блок «Жизнь в эвакуации»

**Результат:** Удалено **~150 строк** закомментированного HTML.

### 6. Перенос инлайн-стилей и скриптов во внешние файлы

**Файлы:** `index.html` → `style.css` / `script.js`

В `index.html` присутствовали:
- Инлайн `<style>` блок (~145 строк) со стилями модального окна «Через огненные вёрсты войны»
- Инлайн `<script>` блок (~30 строк) с логикой открытия/закрытия этого модального окна

**Результат:**
- ~145 строк CSS перенесены в `style.css`
- ~30 строк JS перенесены в `script.js`
- Инлайн-блоки удалены из `index.html`
- Все стили теперь в одном источнике истины

### 7. Исправление robots.txt

**Файл:** `robots.txt`

Директива `Disallow: /assets/` блокировала индексацию всех изображений поисковыми системами, что противоречило записям `<image:image>` в `sitemap.xml`.

**Результат:** Удалена директива `Disallow: /assets/`.

## Рекомендации (не выполнены автоматически)

### Критические

| # | Проблема | Файл | Приоритет |
|---|---------|------|-----------|
| 1 | `sitemap.xml` использует `example.com` как домен | `sitemap.xml` | Высокий |
| 2 | Schema.org JSON-LD в `index.html` также использует `example.com` | `index.html` | Высокий |
| 3 | Динамический header (`renderHeader`) содержит только 3 ссылки, тогда как статический header на `index.html` — 5 ссылок. Навигация на подстраницах неполная. | `script.js`, `index.html` | Высокий |

### Средние

| # | Проблема | Файл | Приоритет |
|---|---------|------|-----------|
| 4 | Кнопка музыки `<audio>` + `<button>` дублируется в каждом из 8 HTML-файлов (~25 строк × 8 = ~200 строк). Можно генерировать через JS. | Все HTML | Средний |
| 5 | Lightbox `<div id="lb-overlay">` дублируется в 4 HTML-файлах. Можно генерировать через JS. | `gallery.html`, `awards.html`, `personal-archive.html`, `media.html` | Средний |
| 6 | Модальное окно «О музее» дублируется в `index.html` и `portnova.html` с разным контентом. | `index.html`, `portnova.html` | Средний |
| 7 | Несколько экземпляров `IntersectionObserver` создаются в разных местах кода. Можно объединить в один общий. | `script.js`, `war-path.html` | Средний |
| 8 | Большинство `<img>` тегов не имеют атрибутов `width`/`height`, что вызывает Cumulative Layout Shift (CLS). | Все HTML | Средний |

### Низкие

| # | Проблема | Файл | Приоритет |
|---|---------|------|-----------|
| 9 | `<noscript>` fallback отсутствует — при отключённом JS галереи и header/footer будут пустыми. | Все HTML | Низкий |
| 10 | Google Fonts загружаются на каждой странице отдельно. Рассмотреть self-hosting. | Все HTML | Низкий |

## Архитектурные замечания

### Положительные стороны проекта

- Отличная доступность: ARIA-метки, skip-link, focus trapping, keyboard navigation
- Адаптивный дизайн с 6 breakpoints
- Хорошая SEO-оптимизация: Open Graph, Twitter Cards, Schema.org
- Тёмная тема с золотым акцентом и glassmorphism эффектами
- Data-driven галереи через JSON в `<script type="application/json">`
- Оптимизация изображений через формат WebP и `loading="lazy"`

### Что можно улучшить в будущем

1. **Единый компонент музыки** — вынести рендер `<audio>` + `<button>` в JS
2. **Единый компонент lightbox** — генерировать через JS при необходимости
3. **Консистентная навигация** — синхронизировать ссылки в `renderHeader()` со всеми страницами
4. **Общий IntersectionObserver** — один наблюдатель на всю страницу
5. **Добавить width/height к изображениям** — для предотвращения CLS
